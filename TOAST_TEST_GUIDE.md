# Toast 메시지 확인 가이드

## 🚀 서비스 실행하기

### 1단계: 터미널 열기
- Windows: `PowerShell` 또는 `명령 프롬프트` 열기
- Mac: `터미널` 열기

### 2단계: 프로젝트 폴더로 이동
```bash
cd C:\Users\user\Desktop\HEALO_Demo
```

### 3단계: 서비스 실행
```bash
npm run dev
```

### 4단계: 브라우저 열기
터미널에 나오는 주소를 복사해서 브라우저에 붙여넣기
- 보통 `http://localhost:5173` 같은 주소가 나옵니다

---

## ✅ Toast 메시지 확인 방법

### 1. 로그아웃 메시지 (가장 쉬움!)

**어디서?**
- 화면 우측 상단에 로그인/로그아웃 버튼이 있는 헤더

**어떻게?**
1. 로그인되어 있다면 → "Log Out" 버튼 클릭
2. **결과**: 우측 상단에 초록색 Toast 메시지 "로그아웃되었습니다!" 표시

---

### 2. 로그인 실패 메시지

**어디서?**
- 로그인 페이지 (`/login`)

**어떻게?**
1. 로그인 페이지로 이동
2. 잘못된 이메일/비밀번호 입력 (예: `test@test.com` / `wrong123`)
3. "Log In" 버튼 클릭
4. **결과**: 우측 상단에 빨간색 Toast 메시지 "로그인에 실패했습니다..." 표시

---

### 3. 회원가입 검증 메시지

**어디서?**
- 회원가입 페이지 (`/signup`)

**어떻게?**
1. 회원가입 페이지로 이동
2. 필수 항목을 비워두고 "Sign Up" 버튼 클릭
3. **결과**: 빨간색 Toast "모든 필드를 입력해주세요." 표시

**다른 테스트:**
- 비밀번호를 5자 이하로 입력 → "비밀번호는 최소 6자 이상이어야 합니다."
- 비밀번호와 확인 비밀번호가 다르게 → "비밀번호가 일치하지 않습니다!"

---

### 4. 문의 폼 검증 메시지

**어디서?**
- 문의 페이지 (`/inquiry`)

**어떻게?**
1. 화면 하단의 "Inquiry" 버튼 클릭 (또는 우측 하단 플로팅 버튼)
2. "Inquiry Form" 선택
3. 필수 항목을 비워두고 "Send Inquiry" 버튼 클릭
4. **결과**: 빨간색 Toast "필수 항목을 모두 입력해주세요." 표시

---

### 5. 관리자 페이지 메시지 (관리자 계정 필요)

**어디서?**
- 관리자 페이지 (`/admin`)

**어떻게?**
1. 관리자 계정으로 로그인 (`admin@healo.com`)
2. 병원 관리 또는 시술 관리 탭으로 이동
3. 병원명 없이 저장 버튼 클릭
4. **결과**: 빨간색 Toast "병원명은 필수입니다." 표시

**성공 메시지 확인:**
1. 모든 정보 입력 후 저장
2. **결과**: 초록색 Toast "병원 정보가 저장되었습니다! 🏥" 표시

## 🎨 Toast 메시지 특징

### 위치
- 화면 **우측 상단**에 표시됩니다
- 여러 개가 동시에 나타날 수 있습니다 (위에서 아래로 쌓임)

### 색상
- 🟢 **초록색**: 성공 (success)
- 🔴 **빨간색**: 에러 (error)
- 🔵 **파란색**: 정보 (info)
- 🟡 **노란색**: 경고 (warning)

### 동작
- 3초 후 자동으로 사라집니다
- X 버튼을 클릭하면 즉시 사라집니다
- 여러 메시지가 있으면 위에서 아래로 쌓입니다

## 🐛 문제 해결

### Toast 메시지가 안 보여요
1. 브라우저를 새로고침해보세요 (F5 또는 Ctrl+R)
2. 개발자 도구(F12)를 열어서 에러가 있는지 확인
3. 터미널에 에러 메시지가 있는지 확인

### 서비스가 실행이 안 돼요
```bash
# 의존성 설치 (처음 한 번만)
npm install

# 그 다음 실행
npm run dev
```
